# EcoReport

## Технологический стек
- **FastAPI**: Основной фреймворк для API.
- **Dishka**: Dependency Injection (DI) контейнер.
- **RabbitMQ**: Брокер сообщений для асинхронной обработки.
- **Pydantic**: Валидация входных данных.
- **Pandas**: Обработка данных для импорта из XLSX/CSV.
- **Aio-pika**: Асинхронная библиотека для работы с RabbitMQ.

## Архитектура
1. **API**: Принимает запросы на генерацию отчетов и ставит их в очередь `reports.generate`.
2. **Worker (ReportWorker)**: Слушает очередь, генерирует PDF-документ (имитация), загружает его в S3-хранилище (имитация) и уведомляет о готовности через очередь `reports.generated`.
3. **Importer**: Позволяет загружать входные данные из различных форматов (Manual, CSV, XLSX) для последующей обработки.

## Установка и запуск
Для запуска проекта необходим установленный Docker.

1. Склонируйте репозиторий.
2. Откройте консоль в корневой директории проекта.
3. Выполните команду:
   ```bash
   docker-compose up -d
   ```
   Это запустит RabbitMQ и само приложение в контейнерах.

## API Endpoints
- `GET /` — Главная страница приложения.
- `POST /generate-report` — Запуск процесса генерации отчета.
  - Параметр: `report_id` (int).

## Проверка работоспособности
После запуска проекта:
1. Перейдите по адресу `http://localhost:8000/`.
2. Чтобы запустить генерацию отчета, можно выполнить POST-запрос:
   ```bash
   curl -X POST "http://localhost:8000/generate-report?report_id=123"
   ```
3. В логах контейнера приложения вы увидите процесс обработки:
   ```bash
   docker-compose logs -f app
   ```
   Вы должны увидеть сообщения о начале генерации, загрузке в S3 и отправке уведомления.

## Модели данных
Система поддерживает валидацию данных для:
- Географических объектов (точки, полигоны).
- Различных типов объектов (болота, городские застройки, ООПТ).
- Результатов химического анализа (выщелачивание).
- Дренажных систем и используемых материалов.
