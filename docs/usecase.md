## 1. Регистрация пользователя

**Предусловие:**  
Пользователь не авторизован в системе.

**Основной сценарий:**
1. Пользователь открывает страницу регистрации.
2. Пользователь вводит имя пользователя.
3. Пользователь вводит пароль.
4. Пользователь нажимает кнопку «Зарегистрироваться».
5. Пользователь автоматически перенаправляется на страницу входа в приложение.

**Альтернативный сценарий:**
- Имя пользователя уже существует - система выводит сообщение об ошибке.
- Ошибка соединения - система выводит сообщение о невозможности завершить операцию.
- Пользователь отменяет регистрацию - возвращение на страницу входа.

**Результат:**  
Создана новая учётная запись, пользователь находится на экране входа.

---

## 2. Вход в систему

**Предусловие:**  
Пользователь зарегистрирован.

**Основной сценарий:**
1. Пользователь открывает страницу входа.
2. Пользователь вводит имя пользователя.
3. Пользователь вводит пароль.
4. Пользователь нажимает кнопку «Войти».
5. Пользователь перенаправляется на страницу "Отчеты".

**Альтернативный сценарий:**
- Введены неверные данные - система выводит сообщение об ошибке.
- Ошибка соединения - система выводит сообщение о невозможности входа.
- Пользователь нажимает на кнопку "Зарегистрироваться" - переход на страницу регистрации.

**Результат:** 
Пользователь авторизован и находится на странице "Отчеты".

---

## 3. Создание отчета

**Предусловие:**  
Пользователь авторизован и находится на странице "Отчеты".

**Основной сценарий:**
1. Пользователь нажимает кнопку с изображением файла в папке (находящуюся на хедере, вторую слева)
2. Открывается страница "Новый отчет".
3. Пользователь заполняет данными поля на странице.
4. Пользователь нажимает кнопку «Сгенерировать отчет».
5. Пользователь перенаправляется на страницу "Отчеты".

**Альтернативный сценарий:**
- Введены неверные данные - система выводит сообщение об ошибке.
- Ошибка соединения - система выводит сообщение об ошибке.

**Результат:** 
Отчет создан и отображается на странице "Отчеты".

---

## 4. Импорт

**Предусловие:**  
Пользователь авторизован и находится на странице "Новый отчет".

**Основной сценарий:**
1. Пользователь нажимает на поле "Прикрепите файл формата .xlsx / .csv ...".
2. Пользователь выбирает файл на своём устройстве.
3. Пользователь подтверждает импорт.
4. Система проверяет корректность файла.
5. Система загружает данные.
6. Импортированные данные подставляются в соответствующие поля на странице "Новый отчет".

**Альтернативный сценарий:**
- Введены неверные данные - система выводит сообщение об ошибке.
- Ошибка соединения - система выводит сообщение об ошибке.

**Результат:** 
Данные отображаются в соответствующих полях на странице "Новый отчет".

## 8. Скачивание отчета

**Предусловие:**  
Пользователь авторизован и находится на странице "Отчеты", существует хотя бы 1 созданный отчет.

**Основной сценарий:**
1. Пользователь нажимает кнопку "Скачать pdf/docx".
2. Система загружает на устройство пользователя файл в формате pdf/docx.

**Альтернативный сценарий:**
- Ошибка соединения - система выводит сообщение об ошибке.

**Результат:** 
Файл в формате pdf/docx загружен на устройство пользователя.

## 10. Изменение данных пользователя

**Предусловие:**  
Пользователь авторизован и находится на странице "Отчеты" или на странице "Новый отчет".

**Основной сценарий:**
1. Пользователь нажимает кнопку с изображением шестеренки (находящуюся на хедере, вторую справа).
2. Открывается страница "Изменение данных пользователя".
3. Пользователь изменяет данные.
4. Пользователь нажимает кнопку "Принять".
5. Система сохраняет новые данные пользователя.

**Альтернативный сценарий:**
- Ошибка соединения - система выводит сообщение об ошибке.

**Результат:** 
Данные пользователя изменены.
