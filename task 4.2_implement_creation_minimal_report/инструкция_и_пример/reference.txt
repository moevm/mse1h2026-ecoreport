1. Необходимые библиотеки
Для работы требуется установить библиотеку ReportLab и jinja2.
Установка через pip: pip install reportlab
Установка через pip: pip install reportlab jinja2

2. Структура проекта
Ожидается следующая структура файлов и папок (все файлы должны быть в кодировке UTF-8):
проект/
│
├── generator.py                  # сам скрипт (название может быть любым)
├── Times New Roman/               # папка со шрифтом (см. раздел 3)
│   └── timesnewromanpsmt.ttf      # файл шрифта Times New Roman
│
└── report_module/                  # папка с исходными данными для отчёта
    ├── content.txt                  # список названий разделов (по одному в строке)
    ├── section_1.txt                 # текст первого раздела (с разметкой)
    ├── section_2.txt                 # текст второго раздела
    └── ...                           # остальные файлы разделов

3. Требования к файлам
3.1. content.txt
Файл содержит названия разделов в том порядке, в котором они должны идти в отчёте.
Каждое название – на отдельной строке. Пустые строки игнорируются.

Пример:
1. ИНФОРМАЦИЯ ОБ ОБЪЕКТЕ
2. ЭКОЛОГИЧЕСКАЯ ПОЛИТИКА
3. ОСНОВНЫЕ ДОКУМЕНТЫ

3.2. Файлы разделов (section_1.txt, section_2.txt, …) файлы с содержимым разделов, использующие разметку и переменные Jinja2.
Имена файлов должны соответствовать шаблону section_{N}.txt, где N – номер раздела, начиная с 1.
Внутри используется простая разметка:
TITLE: – заголовок раздела (всё, что после двоеточия, становится заголовком).
PARA: – обычный абзац (текст после двоеточия).
LIST: – начало маркированного списка. Все последующие строки до пустой строки считаются элементами списка.
Пустая строка разделяет логические блоки.
Любая другая строка (не начинающаяся с маркера) интерпретируется как абзац (без маркера PARA).
----------------------
Пример 
TITLE: 1. ИНФОРМАЦИЯ ОБ ОБЪЕКТЕ
PARA: Настоящий отчет подготовлен в целях анализа...
PARA: Основной задачей настоящего отчета является:
LIST:
систематизация данных
анализ результатов
оценка соответствия

PARA: Организация НАЗВАНИЕ ОРГАНИЗАЦИИ осуществляет...
---------------------

5. Вызов функции из другого модуля
from generator import generate_report

generate_report(user_data)   

После успешного выполнения в текущей рабочей директории появится файл report.pdf.
