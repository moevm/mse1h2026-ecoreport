# Решения по повестке

## 1. В каком виде должно быть разработанное решение

**1.1** Решение представляет собой веб-сервис.

## 2. Основной пользователь

**2.1** Инженер/лаборант, не обладающий продвинутыми навыками работы с
ПК.\
**2.2** Интерфейс должен быть простым и понятным.

## 3. В каком виде должны поступать входные данные

**3.1** Гибридный формат: - ручной ввод
- загрузка `.xlsx` / `.csv` файлов

**3.2** Возможные колонки: - точка отбора (название)
- материал
- широта/долгота
- загрязнители
- и др.

Необходимо предусмотреть работу с табличными файлами с большим
количеством параметров (колонок).

## 4. Проверка корректности вводимых данных

**4.1** Валидация типов данных и диапазонов (числа с плавающей точкой,
допустимые диапазоны значений)\
**4.2** Контроль обязательных полей\
**4.3** Проверка допустимых форматов файлов\
**4.4** Обратная связь пользователю (ошибки и подтверждения)

## 5. Формат отчёта на выводе

**5.1** Минимум: генерация PDF.\
По возможности: выбор формата отчёта (PDF / DOCX).

## 6. Возможность редактировать отчёт

**6.1** См. пункт 5.1

## 7. Соответствие нормативным документам

**7.1** Команде необходимо изучить нормативные документы для обеспечения
соответствия требованиям.

## 8. Пример отчёта

**8.1** Пример отчёта предоставлен.

## 9. Экспорт в GeoJSON

**9.1** Для экспорта в ГИС-системы (QGIS, Яндекс.Карты).\
**9.2** Цветовая гамма точек на карте должна зависеть от значения
экспортируемых данных.

## 10. Первая версия продукта должна включать

1.  Форму для ручного ввода данных и загрузки `.csv` / `.xlsx`
2.  Валидацию данных
3.  Генерацию отчёта и GeoJSON (минимум одна столбчатая диаграмма)
4.  Возможность скачать отчёт и файл GeoJSON

## 11. Ограничения по стеку технологий

**11.1** Ограничений нет. Команда может использовать любые необходимые
инструменты.
